## What is WalletLens?

WalletLens is an AI-powered EVM wallet intelligence tool. Paste any Ethereum, Polygon, BNB Chain, Arbitrum, or Base wallet address and get a full breakdown in seconds:

- **AI Report** — Plain-English behavioral analysis generated by Groq (Llama 3.3 70B)
- **Risk Score** — 0–100 risk gauge based on on-chain behavior patterns
- **Portfolio Breakdown** — Token holdings, USD values, and distribution chart
- **NFT Holdings** — Visual grid of all NFTs in the wallet
- **Transaction History** — Last 20 transactions with status and direction
- **Shareable Reports** — Every analysis gets a unique URL

No wallet connection required. Read-only. Free to use.

## Getting Started

### Prerequisites

- Node.js 18+
- npm or yarn
- Moralis API key (free at moralis.io)
- Groq API key (free at console.groq.com)
- Supabase project (free at supabase.com)

### Installation

```bash
# Clone the repository
git clone https://github.com/theweb3wizard/walletlens.git
cd walletlens

# Install dependencies
npm install

# Copy environment variables
cp .env.local.example .env.local
```

### Environment Variables

Create a `.env.local` file in the root directory:

```env
# Moralis - Blockchain Data
MORALIS_API_KEY=your_moralis_api_key

# Groq - AI Analysis
GROQ_API_KEY=your_groq_api_key

# Supabase - Database
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
```

### Database Setup

Run the following SQL in your Supabase SQL Editor:

```sql
create table analyses (
  id uuid default gen_random_uuid() primary key,
  slug text unique not null,
  wallet_address text not null,
  chain text not null default 'eth',
  token_data jsonb,
  nft_data jsonb,
  transaction_data jsonb,
  net_worth numeric,
  ai_report text,
  risk_score integer,
  wallet_age_days integer,
  is_whale boolean default false,
  created_at timestamp with time zone default now()
);

create table usage_limits (
  id uuid default gen_random_uuid() primary key,
  ip_hash text not null,
  count integer default 1,
  date date default current_date,
  unique(ip_hash, date)
);

-- Permissions
grant all on table analyses to service_role;
grant all on table usage_limits to service_role;
grant usage on schema public to service_role;
```

### Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## How It Works

```
User pastes wallet address
        ↓
Rate limit check (5/day per IP via Supabase)
        ↓
Cache check (return existing if analyzed < 1hr ago)
        ↓
Parallel fetch via Moralis API:
  - Token balances + USD prices
  - NFT holdings
  - Transaction history (last 20)
  - Net worth
  - Wallet age (first transaction date)
        ↓
Groq AI generates intelligence report
  - Wallet overview
  - Portfolio breakdown
  - Behavioral analysis
  - Key insights
  - Risk assessment
  - Risk score (0-100)
        ↓
Result saved to Supabase with unique slug
        ↓
User redirected to /report/[slug]
```
